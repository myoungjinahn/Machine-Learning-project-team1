# **머신러닝 수질 데이터를 활용한 음용가능성 분류** 참고 프로젝트 제안서

## 1.프로젝트 개요

### 주제
**수질 지표 데이터를 활용한 물 샘플의 음용 가능성 분류**

 물의 화학적·물리적 특성을 나타내는 여러 변수를 분석하여, 특정 샘플의 물이 **안전한지(1) or 안전하지 않은지(0)** 를 구분하는 이진 분류(Binary Classification) 문제

### 목표

1. **데이터 탐색 및 전처리**
   - 수질 데이터셋에 포함된 결측치를 적절히 처리하고,각 변수가 수질에 어떤 영향을 미치는지 상관관계를 분석

2. **KNN 알고리즘 적용**
   - 새로운 수질 데이터가 입력되었을 때,기존의 유사한 데이터들과 비교하여 수질을 판정하는 모델 구축

3. **모델 최적화 및 평가**
   - 단순히 모델을 돌리는 것에 그치지 않고,최적의 k값을 찾아내어 모델의 정확도를 향상

4. **수질 판단의 자동화**
   - 사람이 일일이 복잡한 화학 지표를 보고 판단하는 대신,머신러닝 모델을 통해 빠르고 객관적으로 수질의 안전성을 예측할 수 있는 시스템의 가능성 확인


## 2.선택 이유

- 수질·대기·기상 데이터는 대부분 수치형 벡터(pH, 온도, 농도 등) 로 구성되어 있어, 거리를 기준으로 유사한 샘플을 탐색하는 KNN 방식과 자연스럽게 맞음
- 수질 데이터는 물리·화학·생물 반응이 얽혀 그래프에서 직선으로 나타내기 힘든 비선형 관계가 많은데,KNN은 별도의 모델 없이 유연하게 비선형 패턴을 반영 가능
- KNN은 구현이 간단하고 결과 해석이 직관적이어서, 머신러닝 입문 프로젝트로 적합
- Kaggle의 공개 데이터셋을 사용해 재현 가능성이 높고 데이터 접근이 쉬움


## 3.사용 머신러닝 기법

### 알고리즘: K-최근접 이웃 (K-Nearest Neighbors, KNN)

새로운 데이터를 주변의 '가까운 데이터들'을 보고 분류/예측하는 알고리즘

 - 학습 유형: **지도학습 (Supervised Learning)**
 - 모델 유형: **비모수적(Non-parametric)**,**게으른 학습(Lazy Learning)**
 - 사용 형태: **분류(Classification)** — k개 이웃의 다수결로 레이블 결정
 - 거리 척도: **유클리드 거리(Euclidean Distance)**
 - 주요 하이퍼파라미터: 이웃의 수 `k`,거리 척도,스케일링 방식

```python
from sklearn.neighbors import KNeighborsClassifier
knn = KNeighborsClassifier(n_neighbors=5)
```

### 데이터 전처리 기법

1. **결측치 처리 (KNN Imputer)**
   - 수질 데이터에는 비어있는 값(NaN)이 많아, 데이터 삭제 대신 **평균값으로 대체**하는 방식을 선택 (`fillna()` 활용)

2. **데이터 스케일링**
   - 단위 차이가 매우 크므로 (`Solids`: 10000~40000 단위 vs `pH`: 0~14) **StandardScaler**를 이용해 평균 0, 표준편차 1로 표준화

3. **하이퍼파라미터 튜닝**
   - 최적의 이웃 개수 `k`를 찾기 위해 다양한 값을 테스트하며 정확도를 비교

### 상관관계 분석

모델을 만들기 전, 어떤 변수가 수질(`Potability`)과 가장 밀접한 관련이 있는지 파악하는 기법 적용

## 4.데이터셋 설명

### 원본 데이터

 - 출처: Kaggle — "Water Potability"
 - 파일명: `water_potability.csv`
 - 크기: 약 525.19 KB
 - 전체 행 수: 3,276개
 - 컬럼 수: 10개

### 변수 구성

- **독립 변수(Feature)**: 9개의 수치형 수질 성분 데이터
  - `ph`, `Hardness`, `Solids`, `Chloramines`, `Sulfate`, `Conductivity`, `Organic_carbon`, `Trihalomethanes`, `Turbidity`
- **종속 변수(Target)**: 1개의 범주형 변수
  - `Potability` — 식수 적합 여부 (0: 부적합 / 1: 적합)

### 주요 특이사항

- `ph`, `Sulfate`, `Trihalomethanes` 3개 컬럼에 **결측치(NaN) 존재**
- 타겟 변수 `Potability`의 0과 1의 비율이 완벽한 5:5가 아닐 가능성이 있어 클래스 불균형 고려 필요

### Train / Test 분리

 - 분리 방법: `train_test_split` (scikit-learn)
 - 훈련 데이터: 전체의 **80%**
 - 테스트 데이터: 전체의 **20%** (`test_size=0.2`)
 - 훈련 데이터 구조 입력: 9개 수질 성분 / 출력: Potability (0 or 1)
 - 테스트 데이터 역할: 모델이 본 적 없는 데이터로 예측값과 실제 정답 비교

## 5.성능 평가 분석

### 평가 지표

 - **Accuracy**: 전체 예측 중 맞은 비율

### 한계 극복을 위한 추가 시도

- **최적 k값 탐색**: 다양한 `k` 값(예: 1~20)에 대해 정확도를 비교하여 최적값 선택
- **스케일링 적용**: StandardScaler 미적용 시 거리 계산 왜곡 발생 → 전처리 필수
